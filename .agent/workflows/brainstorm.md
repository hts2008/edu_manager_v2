## Steps

1. Read user request. Identify: problem space, constraints, desired outcome.

2. Check `memory/memory-bank/decisionLog.md` for any prior decisions on this topic.

3. Generate ≥ 3 distinct approaches. For each:
   - Name (e.g., "Monolith", "Microservices", "Serverless")
   - 3-5 pros, 3-5 cons
   - Rough effort estimate
   - Key risk

4. Build comparison matrix:
   | Criteria | Approach A | Approach B | Approach C |
   |----------|-----------|-----------|-----------|
   | Complexity | | | |
   | Maintainability | | | |
   | Time to implement | | | |
   | Risk | | | |

5. Present recommendation with justification. If tie → ask user.

6. Document final decision in `memory/memory-bank/decisionLog.md`.
