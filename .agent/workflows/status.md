## Steps

1. Read `KANBAN.md`: count tasks by status (planned, in-progress, implemented, blocked).

2. Read `memory/memory-bank/activeContext.md`: current state, known issues.

3. Read `memory/memory-bank/progress.md`: last 5 entries.

4. Identify blockers: tasks with BLOCKED status, list reasons and unblock paths.

5. Report:
   ```
   Sprint [N]: [X] done, [Y] in progress, [Z] planned, [B] blocked
   Recent: [last 3 completed tasks]
   Blockers: [list + proposed solutions]
   Next: [highest-priority unblocked task]
   ```
