## Steps

1. Elicit requirements: problem, user, acceptance criteria, out of scope.

2. Define data model: entities, relationships, constraints (Mermaid ERD).

3. Define API contract: endpoints, request/response shapes, error codes.

4. Identify edge cases: null, empty, huge, malicious, concurrent.

5. Write ADR for major architectural decisions.

6. Define success metrics: what tests prove this works?

7. Output: spec.md with all sections.
