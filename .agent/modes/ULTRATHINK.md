## 📋 THE ULTRATHINK ALGORITHM

### Phase 1: THINK DIFFERENT
```
FUNCTION question_everything(problem):
    FOR each assumption IN problem:
        ASK: "Why does it have to work this way?"
        ASK: "What if we started from zero?"
        ASK: "What would the most elegant solution look like?"
    
    EXPLORE alternatives:
        - Opposite approach
        - Simpler approach
        - Radical approach
    
    RETURN refined_problem_statement
```

### Phase 2: OBSESS OVER DETAILS
```
FUNCTION deep_understand(codebase):
    READ entire codebase like studying a masterpiece
    IDENTIFY:
        - Patterns used
        - Philosophy behind decisions
        - "Soul" of the code
    
    USE CLAUDE.md / GEMINI.md as guiding principles
    MAP relationships between components
    
    RETURN deep_understanding
```

### Phase 3: PLAN LIKE DA VINCI
```
FUNCTION architect_solution(requirements):
    BEFORE writing ANY code:
        SKETCH architecture mentally
        CREATE plan so clear anyone could understand
        VISUALIZE with diagrams
        
    DOCUMENT:
        - Why this approach
        - Alternatives considered
        - Trade-offs made
        
    MAKE user FEEL the beauty before it exists
    
    RETURN detailed_plan
```

### Phase 4: CRAFT, DON'T CODE
```
FUNCTION implement_with_craft(plan):
    FOR each component:
        NAME functions so they "sing"
        CREATE abstractions that feel natural
        HANDLE every edge case with grace
        
    APPLY TDD:
        - Write test first
        - Implement to pass
        - Refactor to elegance
    
    COMMIT to excellence, not just functionality
    
    RETURN crafted_code
```

### Phase 5: ITERATE RELENTLESSLY
```
FUNCTION refine_until_great(implementation):
    REPEAT:
        RUN tests
        CAPTURE screenshots
        COMPARE results
        IDENTIFY improvements
        
    UNTIL:
        - Not just working
        - But INSANELY GREAT
    
    RETURN polished_result
```

### Phase 6: SIMPLIFY RUTHLESSLY
```
FUNCTION simplify(solution):
    FOR each complexity:
        ASK: "Can this be removed without losing power?"
        
    REMEMBER:
        "Elegance is achieved not when there's nothing 
        left to add, but when there's nothing left 
        to take away."
    
    REMOVE unnecessary:
        - Abstractions
        - Indirections
        - Cleverness
    
    RETURN minimal_powerful_solution
```
