# Review Escalation

## Escalation Levels
- Level 1: Self-resolve (try different approach)
- Level 2: Reroute (assign to different agent)
- Level 3: User input (ask human for decision)
- Level 4: Full stop (blocker requiring external action)

## When To Escalate
- Same error occurs 3 times despite different approaches
- Task requires information only user has
- Risk assessment changes mid-task (discovered higher risk)
- Conflicting requirements discovered
- Resource or access limitation hit

## Escalation Communication
- State what was attempted (specific steps)
- State what failed (specific errors)
- State what is needed (specific ask)
- Propose options if possible
- Include all relevant context and evidence

## Anti-Patterns
- Escalating too early (have not tried alternatives)
- Escalating too late (wasted effort on impossible task)
- Vague escalation (something is wrong - no details)
- Not including context (forcing user to re-investigate)
