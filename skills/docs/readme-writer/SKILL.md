# README Writer

## README Structure
1. Project name + one-line description
2. Badges (CI, coverage, version, license)
3. Key features (bullet list, 3-7 items)
4. Quick Start (3-5 steps to run locally)
5. Installation (detailed, all platforms)
6. Usage examples (common use cases with code)
7. API reference or link to docs
8. Contributing guidelines
9. License

## Writing Tips
- First paragraph should answer: what is this and why should I care?
- Include screenshots or GIFs for visual projects
- Keep Quick Start under 5 commands
- Link to detailed docs instead of cramming everything in README

## Maintenance
- Update on every release
- Keep examples working (test them)
- Remove outdated information promptly
