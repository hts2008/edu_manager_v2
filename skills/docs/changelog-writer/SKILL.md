# Changelog Writer

## Keep a Changelog Format
- Added: new features
- Changed: changes to existing functionality
- Deprecated: soon to be removed features
- Removed: removed features
- Fixed: bug fixes
- Security: vulnerability fixes

## Semver Rules
- MAJOR: breaking changes (remove API, change behavior)
- MINOR: new features, backward compatible
- PATCH: bug fixes, backward compatible

## Writing Good Entries
- User-facing language, not implementation details
- Link to issue/PR for each entry
- Group by impact: breaking first, then features, then fixes

## Release Notes
- Highlight top 3 changes
- Migration guide for breaking changes
- Known issues section
- Upgrade path from previous version
