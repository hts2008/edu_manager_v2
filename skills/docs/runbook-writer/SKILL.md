# Runbook Writer

## Runbook Structure
- Title: what this runbook covers
- Audience: who should use this
- Prerequisites: access, tools, permissions needed
- Steps: numbered, specific, copy-pasteable
- Verification: how to confirm each step worked
- Rollback: how to undo if something goes wrong

## Incident Response
- Detection: what alerts fire
- Assessment: severity classification
- Communication: who to notify
- Mitigation: immediate steps to reduce impact
- Resolution: root cause fix
- Post-mortem: lessons learned template

## Writing Quality
- Assume reader is stressed and sleep-deprived
- No ambiguity: specifc IP, exact command, precise path
- Include expected output for verification
- Test runbook with someone who didnt write it
