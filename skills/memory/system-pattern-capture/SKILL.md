# System Pattern Capture

## What To Capture
- Recurring code patterns specific to this project
- Integration patterns between services
- Error handling patterns that work well
- Testing patterns for specific frameworks
- Deployment patterns for the project infrastructure

## Pattern Format
- Name: descriptive identifier
- Context: when this pattern applies
- Solution: the pattern implementation
- Evidence: where it has been used successfully
- Anti-pattern: what NOT to do

## Storage Location
- memory/brain/learned-patterns.md for project patterns
- memory/brain/system-patterns.md for system-level patterns

## When To Capture
- After solving a difficult problem
- When same solution works across multiple files
- After debugging a tricky issue
- When discovering an effective library usage pattern

## Maintenance
- Review patterns quarterly
- Remove patterns for deprecated technology
- Update patterns when better approach found
