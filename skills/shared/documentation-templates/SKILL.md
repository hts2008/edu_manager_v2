# Documentation Templates

## README Template
- Project name and one-line description
- Prerequisites and quick start (3-5 steps)
- Key features list
- Installation and configuration
- Usage examples with code snippets
- Contributing guidelines and license

## API Endpoint Template
- Method, path, description
- Authentication requirements
- Request parameters with types and examples
- Response format with status codes
- Error responses with codes and messages
- cURL example

## ADR Template
- Title, date, status
- Context (what problem)
- Decision (what was decided)
- Rationale (why this option)
- Alternatives (what else was considered)
- Consequences (positive and negative)

## Runbook Template
- Title and audience
- Prerequisites (access, tools)
- Step-by-step procedure with verification
- Troubleshooting common failures
- Rollback procedure
