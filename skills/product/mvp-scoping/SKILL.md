# MVP Scoping

## MVP Definition
- Minimum: smallest possible scope
- Viable: actually solves the core problem
- Product: usable by real users (not a prototype)

## Feature Prioritization
- Core loop: the one thing users must be able to do
- Remove everything that doesnt serve the core loop
- Ask: if we remove this, does the product still work?

## Scope Cutting Techniques
- Manual before automated (admin panel vs auto-processing)
- Simple before optimized (brute force before caching)
- Single platform before cross-platform
- Invite-only before public launch

## MVP Timeline
- 2-4 weeks for solo developer
- 4-8 weeks for small team
- If longer, scope is too large - cut more
