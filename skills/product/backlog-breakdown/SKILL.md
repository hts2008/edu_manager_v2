# Backlog Breakdown

## Epic to Story
- Epic: large feature (User Authentication)
- Story: user-facing value (As a user, I can login with email)
- Each story independently deliverable and testable

## Story to Task
- Task: developer-level work item (Create login API endpoint)
- Each task under 4 hours
- Clear input/output per task
- Dependencies mapped

## Estimation
- T-shirt sizing for epics: S 1-3d, M 3-7d, L 1-2w, XL 2-4w
- Story points for stories: 1, 2, 3, 5, 8, 13
- Hours for tasks: 1h, 2h, 4h, 8h
- If over 8h, break into smaller tasks

## Backlog Hygiene
- Groom weekly: refine top of backlog
- Remove stale items quarterly
- Keep 2-3 sprints worth of refined stories
